# examples/

Reference patterns and anti-patterns for common design and development scenarios.

## Structure

Each example file should include:
- A clear description of the pattern
- A visual or code example of the **correct** approach
- A visual or code example of the **incorrect** approach (anti-pattern)
- A brief explanation of why the correct approach is preferred

## Naming convention

`{component-or-pattern}-{scenario}.md`

Examples:
- `button-primary-usage.md`
- `form-field-labels.md`
- `empty-state-onboarding.md`
