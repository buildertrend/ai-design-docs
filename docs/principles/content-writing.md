# Content writing

## Intent
Establish a consistent voice and writing standard across all product UI — labels, errors, empty states, tooltips, and any text a user reads in the product.

## Use when
Writing or reviewing any UI copy: button labels, field labels, error messages, empty state text, tooltips, confirmation dialogs, or onboarding text.

## Avoid when
Writing marketing copy, release notes, or long-form documentation — those follow different conventions.

---

## Requirements

### Voice
- **Active voice.** "We couldn't find that search term." Not: "That search term couldn't be found."
- **Second person.** Use "you." Never write from the user's perspective with "my" — e.g. not "My receipts," not "My settings."
- **Contractions.** Use can't, don't, isn't. It reads more naturally.
- **Concise.** Use fewer, shorter words. 1–3 syllable words. Grade 5 reading level or below.
- **Plain language.** No jargon. Ask: if a friend read this without context, would they understand it?
- **Consistent.** Use the same verb across a workflow. Don't mix "link your account," "sync your account," and "add your account."

### Grammar and punctuation
- No exclamation points.
- No emojis — they're inaccessible and contribute to inconsistent tone.
- No period at the end of single sentences or short text strings.
- Periods go inside quotation marks.
- Use the Oxford comma: "Add, edit, and approve."
- No colons in headers or section headers.
- Use "they" instead of "he or she." Prefer "person," "people," "someone," "client," "sub/vendor" over "user."
- Write out full words. Don't use "&" — write "and."

### Spelling
Use Standard American English.
- Color, not colour
- Gray, not grey
- Center, not centre
- Realize, not realise
- Behavior, not behaviour

### Directional language
- Use "select," "choose," "go to," "head to" — never "click" or "tap."
- Never say "here," "above," "left," or "right." Be specific: "Go to folder settings."

---

## Recommendations

- Foreground the most important information. People scan the first 1–3 words and categorize the text — get to the point.
- Write like you speak. If you read it aloud and it sounds convoluted, rewrite it.
- Use inclusive language. Avoid terms like "whitelist," "blacklist," "master," "disable."
- Have someone check your copy, or run it through a grammar tool before shipping.

---

## Terms to avoid

| Instead of | Use |
|---|---|
| Activate / enable | Turn on |
| Administer | Manage / edit |
| Configure | Set up / edit |
| Deactivate / disable | Turn off |
| Display | Show |
| Input | Enter |
| Modify | Edit |
| Notify | Let you know |
| Optimize | Make it better / improve |
| Provide | Give / include / show |
| Purchase | Buy / pay |
| Receive | Get |
| Remain | Stay |
| Retain | Keep |
| Submit | Send |
| Utilize | Use |

---

## Accessibility

- Use inclusive language — avoid terms that assume ability, identity, or device.
- Avoid directional references ("above," "left") that break for screen reader users or different screen sizes.
- Write meaningful button labels — "Save changes," not just "OK."

---

## Anti-patterns

- "Click here" — use descriptive action text.
- "Please enter your..." — drop "please," just say "Enter your..."
- "Successfully saved!" — drop the exclamation point.
- Mixing verbs across a workflow — pick one and stick to it.
- Title Case For Labels — use sentence case.
